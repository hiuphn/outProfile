Cách giảm JavaScript bundle để tăng Core Web Vitals cho website Next.js

Hướng dẫn giảm JavaScript bundle trong Next.js để cải thiện Core Web Vitals, tăng tốc độ tải và giữ trải nghiệm ổn định hơn trên mobile.

Bài này dành cho ai

Bài viết này dành cho người đang dùng Next.js và có cảm giác website “mọi thứ đều ổn” nhưng trải nghiệm trên mobile vẫn nặng nề. Đây là tình huống rất thường gặp ở các site dịch vụ hiện đại: giao diện đẹp, có animation, có interactive cards, nhưng người dùng phải tải quá nhiều JavaScript trước khi có thể tương tác thật sự.

Vì sao JavaScript bundle lại ảnh hưởng mạnh đến Core Web Vitals

Khi trình duyệt phải tải, parse và execute quá nhiều JavaScript, mọi thứ từ thời gian hiển thị nội dung chính đến khả năng phản hồi tương tác đều bị kéo chậm lại. Đó là lý do nhiều website có server tốt nhưng trải nghiệm thực tế vẫn chưa tốt, nhất là trên điện thoại.

1. Rà lại phạm vi của `use client`

Một file có `use client` không chỉ ảnh hưởng riêng file đó. Nó còn có thể kéo theo nhiều dependency khác vào bundle client. Vì vậy, giảm bundle hiệu quả nhất thường bắt đầu từ việc thu nhỏ phạm vi component thật sự cần tương tác.

2. Tách component nặng bằng dynamic import

Không phải component nào cũng cần có mặt ngay ở first paint. Những phần như chart, testimonial động, card tương tác hoặc khối animation dưới viewport có thể tách bằng dynamic import để ưu tiên phần nội dung chính trước.

3. Kiểm tra lại thư viện và assets đi kèm

Bundle lớn không chỉ đến từ code bạn viết, mà còn đến từ các thư viện kéo theo rất nhiều đoạn mã không thực sự cần cho mỗi route. UI library, chart library, editor, map hoặc animation toolkit thường là các nguồn phình bundle rõ nhất.

Một website dịch vụ nhanh hơn thường không đến từ “mẹo tối ưu”, mà đến từ việc bỏ bớt những thứ client không thực sự cần.

Kết luận

Nếu muốn cải thiện Core Web Vitals trong Next.js, giảm JavaScript bundle gần như luôn là một trong những đòn bẩy lớn nhất. Hãy bắt đầu từ `use client`, dynamic import và việc kiểm tra lại các thư viện nặng trước khi tối ưu sâu hơn ở những lớp khác.

Checklist hành động

Kiểm tra những trang nặng nhất và xác định thành phần nào đẩy JS xuống client nhiều nhất

Giảm phạm vi của `use client` và chuyển những khối tĩnh về Server Components

Dùng dynamic import cho phần dưới viewport hoặc component nặng nhưng ít quan trọng

Rà thư viện animation, chart, editor hoặc map để tránh bundle phình quá nhanh
